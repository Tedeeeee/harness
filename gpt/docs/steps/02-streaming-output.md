# 02. 스트리밍 출력 추가

## 이번 단계의 목표

기존 CLI는 Anthropic API의 전체 응답을 모두 받은 뒤 한 번에 출력했다.  
이번 단계의 목표는 이 흐름을 한 단계 확장해서, **모델이 생성하는 텍스트를 도착하는 순서대로 바로 출력하는 스트리밍 방식**을 도입하는 것이다.

즉 이번 단계에서 바꾼 핵심은 다음 한 줄로 정리할 수 있다.

**응답 완료 후 한 번에 출력하던 CLI를, 생성 중인 텍스트를 실시간으로 보여주는 CLI로 바꿨다.**

## 왜 이 단계가 필요한가

단일 호출 CLI는 하네스의 최소 실행 루프를 이해하기에 좋지만, 실제 사용 경험은 아직 다소 답답하다.  
사용자는 모델이 생각을 끝낼 때까지 기다려야 하고, 긴 응답에서는 정상적으로 동작 중인지도 즉시 알기 어렵다.

스트리밍 출력은 다음과 같은 이유로 중요하다.

- 사용자는 응답이 시작되는 순간부터 결과를 볼 수 있다.
- 긴 응답에서도 하네스가 멈춘 것이 아니라 계속 동작 중임을 확인할 수 있다.
- 이후 REPL, 도구 호출, 진행 상태 출력 같은 기능을 붙일 때 UX 기반이 된다.
- 하네스 관점에서 보면, 모델 호출 계층이 응답을 어떻게 소비할 것인가를 더 실제 제품에 가깝게 다루게 된다.

## 이번 단계의 범위

### 포함

- Anthropic API 스트리밍 호출 사용
- 텍스트 조각이 도착할 때마다 CLI에 바로 출력
- 스트리밍 종료 후 줄바꿈 처리
- 스트리밍 중 오류가 발생했을 때 오류 메시지 출력
- 스트리밍 동작을 검증하는 자동 테스트 추가

### 제외

- REPL 루프
- 대화 이력
- 여러 메시지 역할 관리
- 도구 호출
- 세션 저장
- 멀티 에이전트

## 현재 구조와 이번 단계의 변화

기존 구조:

```text
CLI 입력
-> 설정 로드
-> 일반 메시지 호출
-> 전체 텍스트 조립
-> 한 번에 출력
```

변경 후 구조:

```text
CLI 입력
-> 설정 로드
-> 스트리밍 메시지 호출
-> 텍스트 조각 수신
-> 도착 즉시 출력
-> 스트림 종료 후 줄바꿈
```

핵심 차이는 응답을 모두 모은 뒤 출력하던 방식에서, 응답을 받는 즉시 출력하는 방식으로 바뀌었다는 점이다.

## 핵심 구현 내용

### 1. 테스트 가능한 CLI 실행 함수 분리

`src/lib/run-prompt-command.ts`를 새로 추가했다.

이 파일은 다음 책임을 맡는다.

- CLI 인자를 프롬프트 문자열로 결합
- 프롬프트가 없으면 사용법 출력
- 설정 로드
- 스트리밍 호출 실행
- 청크가 들어올 때마다 바로 출력
- 종료 시 줄바꿈 추가
- 오류 발생 시 stderr 출력 및 종료 코드 반환

이 파일을 분리한 이유는 스트리밍 CLI 동작을 자동 테스트로 검증하기 쉽게 만들기 위해서다.

### 2. Anthropic 스트리밍 함수 추가

`src/lib/anthropic.ts`에는 기존 `generateText()`를 유지하면서, 새로 `streamText()`를 추가했다.

`streamText()`는 다음 역할을 맡는다.

- Anthropic SDK의 `messages.stream()` 호출
- `text` 이벤트를 받아 텍스트 조각 전달
- 스트림 종료 대기
- 스트리밍 실패 시 오류 메시지 변환

즉 이 단계에서 모델 호출 계층은 “전체 텍스트 반환” 방식과 “텍스트 조각 전달” 방식을 모두 가지게 됐다.

### 3. CLI 진입점 단순화

`src/cli.ts`는 이제 직접 출력 흐름을 구현하지 않고, `runPromptCommand()`를 호출하는 얇은 진입점 역할만 맡는다.

이렇게 하면 이후 REPL이나 대화 이력을 붙일 때도 진입점은 얇게 유지하고, 실제 흐름 제어는 별도 파일에서 담당할 수 있다.

### 4. 자동 테스트 추가

`tests/streaming-cli.test.ts`에서 아래 세 가지를 검증한다.

- 프롬프트가 없으면 사용법을 출력하는가
- 스트리밍 조각이 순서대로 출력되고 마지막에 줄바꿈이 붙는가
- 스트리밍 중 오류가 발생하면 stderr에 메시지를 출력하는가

이 단계부터는 단순 수동 확인뿐 아니라, 동작 자체를 코드로 고정하기 시작했다는 점이 중요하다.

npm test

## 완료 기준

아래 조건을 모두 만족하면 이번 단계는 완료다.

- Anthropic 스트리밍 호출이 동작한다.
