# Requirements Directory

This directory is for human-authored product requirements.

## Rules

- Treat files here as original input
- Do not overwrite them from planning or implementation flows
- Do not rewrite them into a richer version in place
- If more information is needed, ask questions and record the result elsewhere

## Expected File Style

Requirements should describe:

- goal
- problem
- users
- core features
- out of scope
- constraints
- success criteria

Use `templates/product-requirements-template.md` as a starting point if a human wants help drafting a requirements document.
